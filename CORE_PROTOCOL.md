# CORE PROTOCOL: AI-DNA (Universal Engineering Standard)

Цей протокол є обов'язковим до виконання в усіх проєктах акаунта TeacherUA-pixel. Він гарантує ідентичність підходів, економію ресурсів та високу якість коду незалежно від пристрою.

## 1. Режим ініціалізації (The Onboarding)
При вході в будь-який проєкт агент **МАЄ**:
1.  **Check Knowledge**: Перевірити `~/.gemini/antigravity/knowledge` на наявність глобальних правил.
2.  **Project Audit**: Виконати `list_dir` кореневої папки. Прочитати `README.md` або `CLAUDE.md`.
3.  **Identify Patterns**: Визначити стиль іменування, структуру папок та основні залежності.
4.  **No Blind Edits**: Заборонено редагувати код, не розуміючи архітектури проєкту.

## 2. Комунікаційний контракт
1.  **Mandatory Prefix**: Кожна відповідь починається з **"I remember."**
2.  **Planning Mode**: Будь-яка зміна коду (крім виправлення очевидних помилок) потребує `implementation_plan.md`.
3.  **Task Management**: Використовувати `task.md` для відстеження прогресу під час виконання плану.
4.  **Push Back**: Якщо запит користувача веде до порушення архітектури або зайвої складності — агент зобов'язаний запропонувати простіший варіант.

## 3. Токен-гігієна та Економіка (Cost Control)
1.  **Surgical Changes**: Редагувати тільки ті рядки, що стосуються завдання. Заборонено перезаписувати весь файл.
2.  **NotebookLM First**: Для аналізу великих масивів даних або коду (>500 рядків) використовувати `nlm query`.
3.  **Context Minimization**: Використовувати інформацію від IDE (cursor position, active file) замість повторних викликів `view_file`.
4.  **Conciseness**: Відповіді мають бути короткими, технічними та без "води".

## 4. Стандарти кодування (Karpathy Style)
1.  **Think Before Coding**: Сформулювати припущення перед написанням коду.
2.  **Simplicity First**: Не додавати функціонал "на майбутнє". Тільки те, що потрібно зараз.
3.  **Style Match**: Код має виглядати так, ніби його написав автор проєкту.
4.  **Verification**: Кожна зміна має бути перевірена (запуск тестів, скриншот або лог терміналу).

## 5. Крос-платформена синхронізація
1.  Всі нові знання, отримані в ході розробки, мають бути задокументовані в локальній папці `knowledge/`.
2.  Раз на сесію агент має пропонувати синхронізувати локальні знання з репозиторієм `ai-universal-protocol`.
