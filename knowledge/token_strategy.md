# Глобальна стратегія економії токенів: NotebookLM + Gemini CLI Companion

Цей документ визначає стандарти роботи для забезпечення максимальної ефективності використання токенів у проектах.

## 1. Використання NotebookLM (RAG Пріоритет)
Для будь-якого проекту, що має великі обсяги даних (JSON, великі файли коду, історія цін), **ОБОВ'ЯЗКОВО** використовувати NotebookLM:
- **Замість повного читання файлів**: Використовувати команду `nlm notebook query` для отримання аналітичної інформації.
- **Синхронізація**: Всі важливі оновлення коду та даних мають бути завантажені в блокнот як джерела.
- **Джерела**: Код проекту, документація (`SETUP_HOME.md`), структури баз даних та великі JSON-файли.

## 2. Використання Gemini CLI Companion
Для роботи в IDE (VS Code/JetBrains) **ОБОВ'ЯЗКОВО** використовувати Companion для передачі контексту:
- **Курсор та активний файл**: Використовувати дані про відкритий файл для уникнення зайвих команд `view_file`.
- **Виділення коду**: Працювати з конкретними фрагментами (Selections) замість повних файлів, де це можливо.
- **Native Diffs**: Застосовувати зміни через механізм порівняння компаньйона для зменшення обсягу переданих даних при редагуванні.

## 3. Зв'язка інструментів
- **NotebookLM** = Аналітична пам'ять та робота з даними (RAG).
- **Companion** = Робочий контекст «тут і зараз».
- **Antigravity** = Координатор, який поєднує ці інструменти для виконання завдань.

## 4. Золоте правило
Перед тим як відкрити файл розміром понад 500 рядків, агент **МАЄ** перевірити, чи можна отримати необхідну інформацію через запит до NotebookLM.
