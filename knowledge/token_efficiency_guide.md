# Token Efficiency & Cost Optimization Guide

Цей гід містить практичні методи зменшення витрат токенів при роботі з ШІ-агентами.

## 1. Стратегія Context Pruning
- **Surgical Edits**: Використовувати `multi_replace_file_content` замість повної заміни файлу.
- **Diff-Only Thinking**: Якщо файл великий, агент має бачити лише ті частини, які він редагує.
- **No Redundant Reads**: Заборонено читати один і той самий файл двічі за одну сесію, якщо він не змінювався.

## 2. Використання RAG (NotebookLM)
- **Large Data Handling**: Будь-які JSON або CSV масиви > 100 КБ мають бути завантажені в NotebookLM.
- **Code Reference**: Весь колишній код проєкту має бути проіндексований у блокноті для швидкого пошуку логіки.

## 3. Модельний менеджмент
- **Task Routing**:
    - Важкі задачі (архітектура, складні баги) -> Claude 3.5 Sonnet / GPT-4o.
    - Легкі задачі (тести, коментарі, верстка) -> Haiku / Flash / Small models.
- **Prompt Caching**: Структурування запитів так, щоб статичні частини (правила, протоколи) були на початку (для підтримки кешування на рівні API).

## 4. XML Tagging
- Використання чітких тегів `<context>`, `<requirements>`, `<plan>` для кращого розуміння моделі з меншою кількістю токенів на роздуми.
