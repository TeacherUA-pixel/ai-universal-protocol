# GIT MASTERY: Karpathy-Style & AI-DNA

Цей документ визначає стандарти роботи з Git для забезпечення високої швидкості ітерацій, чистоти історії та токен-ефективності.

## 1. Робочий процес "Ratchet" (Храповик)
Ми розглядаємо розробку як послідовність незворотних покращень.
- **Loop**: Запропонувати зміну -> Запустити перевірку (тести/метрики) -> Оцінити результат.
- **Commit**: Якщо результат покращує систему (метрика зросла) — негайно комітити. Це нова "базова лінія".
- **Reset**: Якщо результат негативний або сумнівний — `git reset --hard` або `git revert`. Ми ніколи не залишаємо "сміття" або непрацюючий код в робочій гілці.

## 2. Хірургічні зміни (Surgical Changes)
AI-агенти повинні мінімізувати шум у дифах:
- Редагувати тільки необхідні рядки.
- Заборонено автоформатування всього файлу, якщо це не було частиною завдання.
- Кожен рядок у коміті має бути обґрунтованим.

## 3. Коміт-повідомлення (Intent over Action)
Коміти мають описувати **намір**, а не просто дію.
- **Погано**: "Updated utils.py"
- **Добре**: "feat: improve bond price calculation precision (fixes #12)"
- **AI Context**: Якщо зміна згенерована AI, корисно додавати `Generated-with: Antigravity` у тіло коміту.

## 4. Токен-ефективне порівняння
- Використовуйте `git diff --stat` для швидкого огляду змін перед глибоким аналізом.
- Запитуйте тільки конкретні частини файлу через `view_file` з вказанням рядків, замість читання всього репозиторію.

## 5. Гігієна репозиторію
- **.gitignore**: Має включати всі специфічні для AI-інструментів папки (`.cursor/`, `.aider.chat.history.md`).
- **Git LFS**: Заборонено комітити великі бінарні файли (>5MB) безпосередньо. Використовуйте Git LFS або зовнішні сховища.
